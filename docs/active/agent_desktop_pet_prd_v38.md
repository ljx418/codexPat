# Agent Desktop Pet PRD V38

Date: 2026-06-26

## Scope

V38 makes the next practical step from document-only photo-to-action planning to a real public-photo sample pipeline. It must use public-authorized cat photos, create sanitized derived images, generate renderable sample-bound 2D action frame packs, expose product UI anchors, and produce screenshot-backed evidence.

V38 is scoped to tested public photo samples. It does not prove arbitrary user photo automation, provider execution, production release, platform readiness, 3D delivery, Petdex parity, MCP readiness, Claude integration, OS-level window binding, or all workflow verification.

## Target User Experience

The user can inspect a real public cat photo sample, see that the app created a metadata-stripped derived image, inspect action contact sheets and GIF previews for idle, walk, jump, sleep, eat, play, alert, and celebrate, and see a product-level gate for preview/apply/rollback readiness.

The user should not see the old simple line cat as the only evidence for this stage. The stage must show image-derived cat assets in an HTML report and product UI status.

## Product Requirements

- Intake at least three different public-authorized cat photos.
- Intake at least one non-cat negative sample and one multi-cat blocked sample.
- Store original downloads only in temporary local storage, not repository evidence.
- Store sanitized derived images in evidence and desktop public assets.
- Strip EXIF/GPS-style metadata from derived images.
- Generate per-sample action frame packs with eight target actions and at least four frames per action.
- Reject transform-only weak motion as sufficient evidence.
- Generate contact sheets, GIF previews, and a readable Chinese HTML report with screenshot evidence.
- Expose V38 settings anchors for public source status, pixel asset status, renderable pack preview, apply/rollback gate, and blocked reason.
- Record Route B as a possible higher-quality comparison path, but execute Route A2 first.

## Exit Criteria

- V38.0 planning audit passes.
- V38.1 public source intake passes with real public-source metadata or stable blocked reason.
- V38.2 sanitized pixel evidence passes.
- V38.3 renderable action pack evidence passes.
- V38.4 quality gate passes scoped or clearly blocks.
- V38.5 product UI contract passes.
- V38.6 HTML report and screenshot evidence pass.
- V38.7 final gate states only a scoped public-photo result.
