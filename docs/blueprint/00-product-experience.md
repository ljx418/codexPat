# 产品体验北极星

## 一句话定位

`agent-desktop-pet` 是一只可常驻桌面的开发者猫，用来把 Codex、Claude Code 和自定义 AI Agent 的后台状态转译为低干扰、可感知、可互动的桌面反馈。

第一版不是通用桌面宠物，不是通知中心，不是监控仪表盘，也不是 3D 角色生成器。第一版要先证明：开发者愿意把这只猫一直放在桌面上，并能通过余光感知 Agent 的任务状态。

## 首批用户

MVP 首批用户是本机开发者：

- 正在使用 Codex、Claude Code 或自定义 agent。
- 能接受本地 CLI、HTTP API 等偏技术化配置。
- 希望在开发过程中获得轻量陪伴和必要提醒。
- 不希望桌面被频繁通知、气泡或声音打断。

## 核心体验

优先级从高到低：

1. 猫咪可常驻桌面，不挡路、不吵闹、不需要用户持续关注。
2. Agent 状态会自然触发猫咪反应，但不会把桌宠变成通知中心。
3. 普通状态只通过低打扰动作让用户余光感知。
4. 只有 `need_input`、`error` 等真正需要用户注意的状态才明显提醒。
5. HTTP API 和 petctl 是接入能力，不是第一感受。

## 状态呈现原则

默认以动作为主，文本为辅：

```text
thinking   -> 背景轻状态，不弹气泡，不播放声音
running    -> 走动或忙碌动作，不弹气泡，不播放声音
success    -> 短暂开心动作，可选轻提示音
warning    -> 轻提醒动作，可选短文本
error      -> 明显错误动作，短文本和声音可用但要克制
need_input -> 强提醒，允许短文本气泡和声音
idle       -> 自主待机
sleeping   -> 睡觉或低活跃状态
```

事件里的 `title` 和 `message` 不代表必须展示。它们是桌面 App 做体验决策的输入，最终是否显示文本由状态机、用户设置和当前打扰级别决定。

## MVP 体验验收

启动体验：

- 启动后 10 秒内看到猫。
- 猫咪窗口透明、无边框、置顶，但不会大面积挡住桌面。
- 可以拖拽猫咪，拖拽后位置被保存。

陪伴体验：

- 无 agent 事件时，猫也会 idle、walk、sleep。
- 鼠标靠近猫咪时有关注反应。
- 用户快速移动鼠标或启用玩具时进入逗猫状态。

Agent 体验：

- `running` 高频事件不会让猫咪反复切状态。
- `success` 有短促正反馈。
- `need_input` 能明显提醒用户。
- 普通状态不频繁弹气泡、不频繁播放声音。

安全体验：

- 外部 agent 只能写结构化事件。
- 不能直接控制 UI、执行脚本、指定本地声音或资源路径。

## 非目标

MVP 不追求：

- 完整 3D 动态猫。
- 照片一键生成 Live2D 或 3D 模型。
- 全局通知中心。
- 强状态仪表盘。
- 完整硬件控制平台。
- 复杂多 agent 时间线。
- 默认点击穿透。
- MCP server、完整 skills、USB 真实或 mock 硬件、照片自定义猫咪、团队协作中枢、自动更新和正式签名发布。
