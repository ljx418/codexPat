# V19 Petdex Resource Boundary

日期：2026-06-12  
状态：planned resource boundary。

## Why Petdex Is Referenced

Petdex is used as a product and format benchmark because it demonstrates:

- A large pet browsing/gallery experience.
- Standardized spritesheet-oriented pet assets.
- Multi-frame action rows with visible motion.
- Clear user expectation for animated desktop pet quality.

Reference links:

- `https://petdex.crafter.run/zh`
- `https://petdex.dev/api/manifest`
- `https://github.com/crafter-station/petdex`

## What We May Use

Allowed:

- Publicly described format ideas.
- Public manifest field analysis.
- Motion-sheet metrics as quality benchmarks.
- User experience patterns such as browse, preview, install, and animated rows.

## What We Must Not Use Without Further Evidence

Forbidden unless explicit license/authorization exists:

- Bundling Petdex user-submitted spritesheets.
- Redistributing Petdex community assets.
- Claiming Petdex asset reuse authorization.
- Treating Petdex sample files as project-owned fixture assets.
- Shipping Petdex-derived visual content.

## V19 License Policy

V19 assets must come from:

1. Project-authored generated assets.
2. User-provided assets with explicit local import consent.
3. Provider-generated assets with consent and license/attribution evidence.
4. Public assets only when license clearly permits the exact project use.

## V19 Claim Boundary

V19 can say "Petdex-style" only to describe compatible sheet structure and product-quality motion goals. It must not say "Petdex parity achieved" unless a separate future evidence gate proves comparable visual quality, gallery depth, install UX, and license-clean ecosystem behavior.
