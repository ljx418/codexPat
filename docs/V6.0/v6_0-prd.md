## Product Position

V6 turns the accepted local engineering baseline into a productized macOS developer workflow.

The productized path must let users:

- install and start the app with clear local security boundaries.
- create a Codex work-cat from Desktop Manager.
- understand JSONL monitor, managed TUI hooks, and already-open window limitations.
- import, preview, activate, roll back, delete, and rename cat asset packs.
- generate local prompt/import instructions for personalized assets without default upload.
- use diagnostics and evidence-backed troubleshooting without leaking sensitive data.

## Current Baseline

V3.x, V4.x, and V5.x are closed scoped baselines:

- V3.7 remains the reliable wrapper-launched `codex exec --json` monitor baseline.
- V4.x remains scoped; OS-level discovery is not lifecycle monitoring.
- V5.0-V5.15 and V5.x Productization Gate passed for scoped local productization.

V6 uses V5.12-V5.15 as accepted baseline evidence. V6 does not reclassify those phases as unfinished work.
